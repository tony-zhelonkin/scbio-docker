# Submodule Setup Instructions

## Add SciAgent-toolkit as Submodule

Run this command in the scbio-docker repository root:

```bash
cd "/Users/tony/Bioinf GDrive/Projects/scbio-docker"
git submodule add git@github.com:tony-zhelonkin/SciAgent-toolkit.git .sciagent
```

If you encounter SSH authentication issues, you have two options:

### Option 1: Use HTTPS (Simpler)

```bash
git submodule add https://github.com/tony-zhelonkin/SciAgent-toolkit.git .sciagent
```

### Option 2: Fix SSH Keys (Recommended for frequent use)

1. Check if you have SSH keys:
```bash
ls -la ~/.ssh/
```

2. If no keys exist, generate them:
```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

3. Add key to ssh-agent:
```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

4. Add public key to GitHub:
```bash
cat ~/.ssh/id_ed25519.pub
# Copy the output and add to GitHub Settings â†’ SSH Keys
```

5. Then retry the submodule add command

## Initialize Submodule

After adding the submodule, initialize it:

```bash
git submodule init
git submodule update
```

## Verify Submodule

Check that the submodule was added correctly:

```bash
git submodule status
ls -la .sciagent/
```

You should see the SciAgent-toolkit files in the `.sciagent/` directory.

## Commit the Submodule

```bash
git add .gitmodules .sciagent
git commit -m "Add SciAgent-toolkit as git submodule for AI tool integration"
```
